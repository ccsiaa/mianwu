## 一、用户洞察与问题定义

### 目标用户

正在求职的应届毕业生和0-3年职场新人，年龄22-28岁。他们面临高频面试压力，但缺乏有效的复盘手段——面试结束后只能凭模糊记忆回顾，不知道自己回答质量如何、哪些地方需要改进。

### 用户痛点

| 痛点 | 说明 |
|------|------|
| 记录困难 | 面试后只能凭模糊记忆复盘，遗漏关键问题和细节 |
| 缺乏反馈 | 不知道回答好坏，无法获得专业评估和改进建议 |
| 经验零散 | 面试经验无法系统沉淀，每次面试都是"从零开始" |
| 简历通用 | 简历内容千篇一律，与目标岗位JD匹配度低 |

### 使用场景

**场景一：面试即时复盘**
面试结束后，用户打开面悟，上传录音或实时转写。系统自动生成对话文本，AI提取关键问题并评估回答质量，给出具体改进建议。用户清晰知道自己哪里回答得好、哪里需要提升。

**场景二：备战新面试**
用户收到某公司面试邀请，打开知识库查看该岗位类型的历史面试问题，回顾自己之前的回答和改进建议，针对性准备，避免重复犯错。

### 产品概述

面悟是一款AI驱动的面试复盘与简历生成平台。通过语音识别自动记录面试内容，AI智能分析评估回答质量，将面试问题沉淀到知识库与个人经历关联，最终基于JD智能生成定制化简历，帮助求职者一站式高效备战。

### 核心功能

| 功能 | 解决的问题 |
|------|------------|
| 语音识别转写 | 解决面试记录困难，自动转写并分离说话人 |
| AI智能分析 | 解决缺乏反馈痛点，提取问题、评级、给出改进建议 |
| 知识库沉淀 | 解决经验零散痛点，问题自动关联到相关经历 |
| 智能简历生成 | 解决简历通用痛点，基于JD匹配生成定制内容 |

## 三、AI原生能力说明

### AI如何解决痛点

**去掉AI，产品还能成立吗？不能。**

| 功能模块 | 无AI时的困境 | AI带来的价值 |
|----------|--------------|--------------|
| 语音转写 | 人工逐字记录，耗时且易遗漏 | 自动转写，1分钟音频1秒完成 |
| 问题分析 | 无法即时评估回答质量 | AI理解语义，即时给出专业反馈 |
| 经历匹配 | 人工逐一比对，效率极低 | 语义匹配，自动关联相关经历 |
| 简历生成 | 手动编写，千篇一律 | JD匹配，针对性输出 |

### AI技术方案

**语音识别层**
- 模型：FunASR Paraformer-large（中文语音识别）
- 说话人分离：speech_campplus_sv_zh-cn（区分不同说话人）
- 处理策略：长音频分段处理（3分钟/段），避免内存溢出

**LLM分析层**
- 模型：Claude API / GPT API
- 应用场景：
  - 问题提取：从转写文本中识别面试问题
  - 分类评级：判断问题类型（项目/基础/行为）和回答质量（良好/一般/待改进）
  - 改进建议：针对每个问题给出具体提升方向
  - 经历匹配：将问题语义匹配到用户相关经历
  - 简历生成：分析JD关键词，匹配经历库，输出定制描述

**工作流程**
```
音频输入 → FunASR转写 → 结构化文本 → LLM分析 → 知识库存储 → LLM简历生成 → 输出
```
