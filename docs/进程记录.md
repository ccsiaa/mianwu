## 2026-05-18 当前版本改动

### 1. 聊天接口改为流式输出

**背景**：原版聊天接口为普通同步请求，等待整段回复后才渲染，体验卡顿。

**改动文件**：

#### `backend/app/api/interview.py`
- `/chat` 接口从普通 `async def` 改为返回 `StreamingResponse`
- 改用 `interview_service.chat_stream()` 替代原来的 `interview_service.chat()`
- 响应格式改为 `text/event-stream`，每个 chunk 以 `data: {"content": "..."}` 形式推送

#### `backend/app/services/interview_service.py`
- 新增 `chat_stream()` 异步生成器方法，支持流式返回 LLM 内容
- 将原来 `chat()` 方法与新 `chat_stream()` 共用的 prompt 构建逻辑抽取为 `_build_chat_system_prompt()` 私有方法
- 新增 `chat_stream_simple()` 通用流式封装（在 llm_service 层）
- 调整各接口 token 上限：
  - `chat` / `chat_stream`：`max_tokens` 从 2000 降至 **800**（对话场景够用，节省资源）
  - `generate_prep_plan`：`max_tokens` 从 4000 降至 **2000**
  - `analyze_interview`：`max_tokens` 从 16000 降至 **6000**
- `chat()` 方法的 `max_tokens` 默认值从 2000 降至 **800**

#### `backend/app/services/llm_service.py`
- `chat()` 同步方法默认 `max_tokens` 从 16000 降至 **2000**
- `chat_stream()` 默认 `max_tokens` 从 2000 降至 **800**
- 新增 `chat_stream_simple()` 包装方法（供外部统一调用流式对话）

#### `src/lib/api.js`
- 新增 `chatInterviewStream()` 函数，使用原生 `fetch` + `ReadableStream` 读取 SSE 流
- 支持回调：`onChunk(chunk)`、`onDone()`、`onError(err)`
- 自动携带 Authorization token，兼容开发/生产环境的 `VITE_API_BASE_URL`

#### `src/pages/InterviewPrep.jsx`
- 将 `chatInterview` 替换为 `chatInterviewStream`
- 发送消息时立即插入空占位 assistant 消息，流式 chunk 到来时逐段追加内容（`setMessages` 更新末尾消息）
- 上下文构建精简：JD 内容截取前 500 字符，只传入准备计划概要和核心技术点（减少 token 消耗）

### 2. 数据库从 SQLite 迁移到 PostgreSQL

**背景**：SQLite 不适合 Render 生产环境的持久化需求，迁移至 PostgreSQL。

#### `backend/app/core/config.py`
- `DATABASE_URL` 默认值从 `sqlite+aiosqlite:///./mianwu.db` 改为 `postgresql+asyncpg://localhost/mianwu`
- `SECRET_KEY` 和 `JWT_SECRET_KEY` 从硬编码字符串改为 `secrets.token_hex(32)` 随机生成

#### `backend/requirements.txt`
- 移除 `aiosqlite==0.19.0`
- 新增 `asyncpg==0.29.0`（PostgreSQL 异步驱动）
- 新增 `psycopg2-binary==2.9.9`（PostgreSQL 同步驱动，供迁移/工具脚本使用）

---

### 3. Render 部署配置更新

#### `backend/render.yaml`
- 新增 `databases` 块，创建 Render 托管 PostgreSQL 实例（`mianwu-db`，香港区域）
- 后端服务部署区域从 `singapore` 改为 `hongkong`（与数据库同区，降低延迟）
- `DATABASE_URL` 环境变量改为从 Render 数据库实例自动注入（`fromDatabase.connectionString`）
- 新增 `SECRET_KEY` 环境变量，由 Render 自动生成（`generateValue: true`）

- `outputDirectory` 从 `build`（CRA 风格）改为 `dist`（Vite 默认输出目录），修复 Vercel 部署找不到产物的问题
